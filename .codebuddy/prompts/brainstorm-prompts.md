# 头脑风暴阶段提示词库

> 🎯 BMAD 方法论 - Brainstorm 阶段常用提示词

---

## 1. 项目启动

### 1.1 新项目探索

```
我正在开始一个新项目：[项目简述]

请作为分析师，帮我进行头脑风暴：

1. 这个项目要解决的核心问题是什么？
2. 目标用户是谁？他们的主要痛点是什么？
3. 有哪些类似的产品或解决方案？它们的优缺点？
4. 我们可以从哪些角度创新或差异化？
5. 这个项目的成功标准应该是什么？
6. 有哪些潜在的风险和挑战？

请帮我全面分析，不要急于给出结论。
```

### 1.2 问题深入分析

```
我想深入分析这个问题：[问题描述]

请帮我从以下角度分析：

1. 问题的根本原因是什么？（5 Why 分析）
2. 这个问题影响了哪些利益相关者？
3. 目前有哪些解决方案？为什么不够好？
4. 如果不解决这个问题，会有什么后果？
5. 理想的解决方案应该是什么样的？
```

### 1.3 目标用户分析

```
我的产品目标用户是：[用户描述]

请帮我深入分析：

1. 这类用户的典型特征是什么？（年龄、职业、技术水平等）
2. 他们在什么场景下会使用我的产品？
3. 他们最大的痛点和需求是什么？
4. 他们目前如何解决这些问题？
5. 他们会为什么样的解决方案付费？
6. 如何触达这些用户？
```

## 2. 竞品分析

### 2.1 竞品全面分析

```
请帮我分析以下竞品：[竞品列表]

分析维度：
1. 产品定位和目标用户
2. 核心功能和特色
3. 优势和劣势
4. 商业模式
5. 用户评价和口碑
6. 我们可以学习什么？
7. 我们可以差异化的点在哪里？
```

### 2.2 功能对比分析

```
请帮我做一个功能对比分析：

我的产品：[产品描述]
竞品A：[竞品A描述]
竞品B：[竞品B描述]

请从以下维度对比：
1. 核心功能完整度
2. 用户体验
3. 性能和稳定性
4. 价格和性价比
5. 生态和集成能力

并输出一个对比表格。
```

## 3. 方案探索

### 3.1 解决方案头脑风暴

```
针对这个问题：[问题描述]

请帮我想出至少 5 种不同的解决方案，包括：
1. 最传统/保守的方案
2. 最创新/激进的方案
3. 最省成本的方案
4. 最好用户体验的方案
5. 最快上线的方案

对于每个方案，请说明：
- 方案描述
- 优点
- 缺点
- 预估成本（时间/人力/资金）
- 主要风险
```

### 3.2 方案可行性评估

```
我有这样一个方案：[方案描述]

请帮我评估可行性：

1. 技术可行性
   - 需要什么技术栈？
   - 有哪些技术难点？
   - 团队是否有相关经验？

2. 商业可行性
   - 市场规模如何？
   - 用户是否愿意付费？
   - 竞争格局如何？

3. 资源可行性
   - 需要多少开发时间？
   - 需要多少人力？
   - 需要多少资金？

4. 风险评估
   - 主要风险有哪些？
   - 如何规避或降低风险？
```

## 4. 范围定义

### 4.1 MVP 范围定义

```
基于之前的讨论，我想定义 MVP 范围。

产品目标：[目标描述]
时间约束：[时间限制]
资源约束：[资源限制]

请帮我：
1. 列出所有可能的功能点
2. 使用 MoSCoW 方法分类（Must/Should/Could/Won't）
3. 给出推荐的 MVP 范围
4. 说明为什么这些功能是 MVP 必须的
5. 后续版本可以加入哪些功能
```

### 4.2 功能优先级排序

```
我有以下功能需求：
[功能列表]

请帮我用 RICE 评分模型排列优先级：
- Reach（影响范围）
- Impact（影响程度）
- Confidence（信心程度）
- Effort（开发工作量）

并给出最终的优先级排序建议。
```

## 5. 约束识别

### 5.1 全面约束分析

```
我的项目情况如下：
- 团队规模：[X]人
- 时间限制：[时间]
- 预算限制：[预算]
- 技术背景：[技术栈]

请帮我识别：
1. 可能面临的技术约束
2. 可能面临的时间约束
3. 可能面临的资源约束
4. 可能面临的业务约束
5. 这些约束对方案选择的影响
6. 如何在约束下做出最优选择
```

---

## 6. 总结输出

### 6.1 头脑风暴总结

```
基于我们之前的讨论，请帮我总结：

1. 核心问题定义
2. 目标用户画像
3. 关键需求列表
4. 推荐解决方案
5. MVP 范围
6. 主要风险和应对
7. 下一步行动计划

请用结构化的格式输出，方便后续使用。
```

### 6.2 生成需求概要文档

```
请根据我们的头脑风暴结果，生成一份需求概要文档，包括：

1. 项目背景
2. 问题陈述
3. 项目目标
4. 目标用户
5. 核心功能（MVP）
6. 成功指标
7. 约束条件
8. 风险评估
9. 下一步计划

这份文档将用于后续的需求映射（Map）阶段。
```

---

## 使用说明

1. **选择合适的提示词**：根据当前讨论阶段选择对应的提示词
2. **填充具体内容**：将 `[方括号]` 中的内容替换为实际项目信息
3. **迭代优化**：根据 AI 回复继续深入讨论
4. **保存记录**：将重要的讨论结果保存到文档中

---

*提示：这些提示词可以根据实际项目需求进行调整和组合。*
