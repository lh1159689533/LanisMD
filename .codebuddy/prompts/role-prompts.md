# 角色专用提示词库

> 🎭 BMAD 方法论 - 各角色专用提示词

---

## 1. 🔍 分析师 (Analyst) 提示词

### 1.1 业务分析

```
作为业务分析师，请帮我分析：

业务领域：[领域描述]
分析目标：[目标]

请提供：
1. 业务现状分析
2. 痛点识别
3. 机会发现
4. 可行性评估
5. 建议方案
```

### 1.2 竞品分析

```
作为分析师，请帮我进行竞品分析：

我的产品：[产品描述]
竞品列表：[竞品1, 竞品2, ...]

分析维度：
1. 市场定位
2. 核心功能
3. 优劣势对比
4. 商业模式
5. 差异化机会
```

### 1.3 需求收集

```
作为分析师，请帮我收集和整理需求：

项目背景：[背景]
利益相关者：[相关者列表]

请：
1. 设计需求收集问题清单
2. 识别显性和隐性需求
3. 对需求进行分类整理
4. 初步评估可行性
```

## 2. 🎯 产品经理 (PM) 提示词

### 2.1 需求定义

```
作为产品经理，请帮我定义需求：

功能描述：[描述]
目标用户：[用户]

请提供：
1. 完整的用户故事
2. 详细的验收标准
3. 优先级评估
4. 依赖关系
5. 风险识别
```

### 2.2 PRD 编写

```
作为产品经理，请帮我编写 PRD：

产品名称：[名称]
产品定位：[定位]
核心功能：[功能列表]

PRD 需要包含所有标准章节。
```

### 2.3 优先级决策

```
作为产品经理，请帮我做优先级决策：

待排序的功能：
[功能列表]

决策依据：
- 业务价值
- 用户需求紧迫度
- 技术复杂度
- 资源约束

请使用 MoSCoW 或 RICE 方法给出排序建议。
```

## 3. 📋 产品负责人 (PO) 提示词

### 3.1 Backlog 管理

```
作为产品负责人，请帮我管理产品待办列表：

当前 Backlog：
[待办项列表]

请：
1. 评估各项的价值
2. 重新排列优先级
3. 识别需要拆分的大项
4. 标记阻塞项
5. 规划版本分配
```

### 3.2 版本规划

```
作为产品负责人，请帮我规划产品版本：

产品目标：[目标]
可用功能列表：[功能]
时间约束：[时间]

请规划：
1. MVP 范围
2. 各版本功能分配
3. 里程碑时间线
4. 依赖和风险
```

### 3.3 需求变更评估

```
作为产品负责人，请帮我评估需求变更：

原始需求：[原需求]
变更请求：[新需求]

请评估：
1. 变更影响范围
2. 对当前 Sprint 的影响
3. 资源需求变化
4. 建议的处理方式
```

## 6. 📋 Scrum Master 提示词

### 6.1 Sprint 规划

```
作为 Scrum Master，请帮我规划 Sprint：

Sprint 时长：[X] 周
团队容量：[X] 人天
待办列表：[Backlog]

请：
1. 计算团队速度
2. 选择 Sprint 目标
3. 分配用户故事
4. 识别风险
5. 制定计划
```

### 6.2 站会主持

```
作为 Scrum Master，请帮我总结今日站会：

成员更新：
[成员1]: [昨天完成/今天计划/阻碍]
[成员2]: [昨天完成/今天计划/阻碍]
...

请：
1. 总结整体进度
2. 识别阻碍
3. 协调资源
4. 提出建议
```

### 6.3 回顾会议

```
作为 Scrum Master，请帮我组织回顾会议：

Sprint 概况：
- 完成的故事：[列表]
- 未完成的故事：[列表]
- 遇到的问题：[列表]

请引导讨论：
1. 做得好的（继续保持）
2. 需要改进的
3. 行动项
```

## 9. 🔍 代码审查员提示词

### 9.1 代码审查

```
作为代码审查员，请审查以下代码：

```[语言]
[代码]
```

审查维度：
1. 功能正确性
2. 代码质量
3. 安全性
4. 性能
5. 可维护性

请指出问题并给出改进建议。
```

### 9.2 安全审查

```
作为代码审查员，请进行安全审查：

代码/模块：[描述]
安全关注点：[关注点]

请检查：
1. 输入验证
2. 认证授权
3. 数据保护
4. 常见漏洞（SQL 注入、XSS 等）
```

---

## 使用说明

1. 根据当前工作角色选择对应的提示词
2. 填充具体的项目信息
3. 可以组合多个角色的提示词
4. 保持角色的一致性和专业性

---

*提示：BMAD 的核心是角色分离，每个角色聚焦自己的专业领域。*
