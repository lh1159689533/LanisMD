# 开发实现阶段提示词库

> 💻 BMAD 方法论 - Develop 阶段常用提示词

---

## 1. 功能实现

### 1.1 功能开发

```
我需要实现以下功能：
[粘贴用户故事和验收标准]

技术背景：
- 项目类型：[Web/Mobile/API...]
- 技术栈：[具体技术栈]
- 现有代码结构：[简述]

请作为开发者，帮我：

1. 分析实现思路
2. 设计代码结构
3. 实现核心代码
4. 编写单元测试
5. 说明需要注意的边界情况

请确保代码：
- 遵循最佳实践
- 包含必要的错误处理
- 有适当的注释
```

### 1.2 API 实现

```
请帮我实现以下 API：

API 规格：
- 方法：[GET/POST/PUT/DELETE]

技术栈：[后端框架]

要求：
1. 完整的请求处理逻辑
2. 参数验证
3. 错误处理
4. 适当的日志记录

```

### 1.3 组件开发

```
请帮我开发以下 UI 组件：

组件名称：[名称]

## 5. 代码审查

### 5.1 代码审查请求

```
请作为高级开发者，审查以下代码：

```[语言]
[代码内容]
```

请从以下维度审查：

1. **功能正确性**：代码是否实现了预期功能？
2. **代码质量**：是否清晰、简洁、可维护？
3. **性能**：是否有性能问题或优化空间？
4. **安全性**：是否有安全漏洞？
5. **测试覆盖**：测试是否充分？
6. **最佳实践**：是否遵循语言/框架最佳实践？

请指出问题并提供改进建议。
```

### 5.2 PR 审查

```
请帮我审查这个 PR：

PR 描述：[描述]
修改的文件：[文件列表]
代码变更：[代码 diff]

请检查：
1. 代码是否符合需求
2. 代码质量
3. 测试覆盖
4. 是否有潜在问题
5. 是否需要文档更新
```

## 10. 总结

### 10.1 开发完成检查

```
我完成了以下功能的开发：

功能描述：[描述]
相关用户故事：[US-XXX]

请帮我检查：
1. 功能是否完整实现

3. 测试覆盖
4. 文档更新

```

### 10.2 Sprint 回顾技术总结

```

1. 做得好的地方
2. 需要改进的地方
3. 学到的经验
4. 下个 Sprint 的建议
```

---

## 使用说明

1. 在完成 Architect 阶段后使用这些提示词
2. 开发前先分解任务，评估工作量
3. 遵循 TDD，先写测试再实现
4. 完成后进行代码审查
5. 保持代码整洁和文档更新

---

*提示：开发过程中保持小步快跑，频繁提交，及时沟通。*
