# 执行计划 (Executing Plans)

## 概述

按批次执行实施计划，在每个批次之间进行审查。

**核心原则：** 批量执行 + 严格按计划 + 每批次审查

## 启动宣言

```
"我正在使用 executing-plans 技能来实施此计划。"
```

## 执行流程

### 第一步：加载与审查计划

1. 读取计划文件
2. **批判性审查** - 识别任何问题或疑虑
3. **如有疑虑**：在开始前向用户提出
4. **如无疑虑**：创建 TodoWrite 并继续

### 第二步：执行批次

- **默认批次**：前 3 个任务
- 对每个任务执行：
  1. 标记为 `in_progress`
  2. **严格按照步骤执行**（计划包含微小的步骤）
  3. 按要求运行验证
  4. 标记为 `completed`

### 第三步：报告

批次完成后：
- 展示已实施的内容
- 展示验证输出
- 说："Ready for feedback."

### 第四步：继续

根据反馈进行操作：
- 如有需要，应用更改
- 执行下一批次
- 重复直到完成

### 第五步：完成开发

所有任务完成并验证后：
1. 宣布："我正在使用 finishing-a-development-branch 技能来完成此工作。"
2. 使用 `finishing-a-development-branch` 技能

## 何时停止并寻求帮助

**立即停止执行**，当发生以下情况时：

- 批次中途遇到阻碍（缺少依赖、测试失败、指令不清）
- 计划存在导致无法开始的关键缺陷
- 你不理解某个指令
- 验证反复失败

**核心原则：** 寻求澄清，不要猜测。

## 关键指导原则

| 原则 | 说明 |
|------|------|
| 首先批判性审查 | 发现问题立即提出 |
| 严格按计划执行 | 不要自作主张修改 |
| 不跳过验证 | 每个步骤都要验证 |
| 批次间只报告等待 | 不要自动继续 |
| 遇阻停止 | 不要猜测 |

## 常见错误

| 错误 | 修正 |
|------|------|
| 自动执行所有任务 | 每 3 个任务暂停一次 |
| 跳过验证步骤 | 每个步骤都要运行验证命令 |
| 遇到问题继续执行 | 立即停止并报告 |
| 修改计划内容 | 严格按计划执行，有问题先沟通 |

## 危险信号

- 一次执行超过 3 个任务
- 跳过测试运行
- 遇到错误继续执行
- 自行决定修改计划

## 与其他技能的集成

- **前置技能**：`writing-plans`
- **后续技能**：`finishing-a-development-branch`
- **替代技能**：`subagent-driven-development`（同会话快速迭代）
