## 背景

我们需要为 TodoApp MVP 选择合适的技术栈。团队规模小（1-2 人），时间紧（4 周），需要快速开发和迭代。

### 约束条件

- 开发周期：4 周
- 团队技术背景：JavaScript/TypeScript
- 部署环境：单服务器
- 预算：有限

---

## 决策

**我们决定**：使用 React + Express + SQLite 技术栈

### 具体选型

| 层次 | 选择 |
|------|------|
| 前端框架 | React 18 + TypeScript |
| UI 样式 | Tailwind CSS |
| 后端框架 | Express 4 + TypeScript |
| 数据库 | SQLite 3 |
| ORM | Prisma |

---

## 理由

### 1. 前端：React + TypeScript

**选择原因**：
- 团队熟悉 React
- TypeScript 提供类型安全
- 生态丰富，组件库多
- 开发效率高

### 2. 样式：Tailwind CSS

**选择原因**：
- 快速开发，无需写 CSS 文件
- 实用优先，减少样式冲突
- 响应式设计方便

### 3. 后端：Express + TypeScript

**选择原因**：
- 轻量级，启动快
- 前后端语言统一
- 灵活，没有过多约束

### 4. 数据库：SQLite

**选择原因**：
- 无需安装数据库服务
- 适合小型应用
- 便于本地开发
- 后续可迁移到 PostgreSQL

### 5. ORM：Prisma

**选择原因**：
- TypeScript 友好
- 类型安全
- Schema 定义清晰
- 迁移方便

---

## 考虑的替代方案

### 方案 A：React + Express + SQLite（已选择）

**优点**：
- ✅ 简单，快速开发
- ✅ 团队熟悉
- ✅ 成本低

**缺点**：
- ❌ SQLite 不适合大规模并发
- ❌ 功能相对基础

**评估**：⭐⭐⭐⭐⭐

---

### 方案 B：Next.js 全栈

**优点**：
- ✅ 一体化框架
- ✅ SSR 支持
- ✅ 部署简单（Vercel）

**缺点**：
- ❌ 学习曲线
- ❌ 对于简单 CRUD 过于复杂

**评估**：⭐⭐⭐⭐

**放弃原因**：MVP 功能简单，不需要 SSR，使用 Next.js 有些过度

---

### 方案 C：Vue + Fastify

**优点**：
- ✅ Vue 开发体验好
- ✅ Fastify 性能高

**缺点**：
- ❌ 团队对 Vue 不够熟悉
- ❌ 需要学习时间

**评估**：⭐⭐⭐

**放弃原因**：团队不熟悉 Vue，会增加开发时间

## 后果

### 正面影响

- ✅ 快速启动开发
- ✅ 团队无学习成本
- ✅ 易于维护和迭代
- ✅ 后续迁移方便

### 负面影响

- ⚠️ SQLite 并发能力有限（MVP 阶段可接受）
- ⚠️ Express 功能较基础（可通过中间件扩展）

### 技术债务

- 后续需要迁移到 PostgreSQL
- 可能需要添加更多中间件
