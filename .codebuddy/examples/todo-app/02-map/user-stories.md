## US-001: 创建任务

### 基本信息

| 属性 | 内容 |
|------|------|
| 故事 ID | US-001 |
| 标题 | 用户可以创建新任务 |
| 优先级 | Must Have |
| 故事点 | 3 |
| 状态 | Ready for Dev |

### 故事描述

**作为** 用户，

**我想要** 快速创建新任务，

**以便** 记录我需要完成的事项，不会遗忘。

### 验收标准

#### AC1: 成功创建任务

```gherkin
Given 用户在任务列表页面
  And 输入框为空
When 用户在输入框中输入任务标题 "买牛奶"
  And 按下回车键或点击添加按钮
Then 新任务 "买牛奶" 出现在任务列表顶部
  And 输入框被清空
  And 任务状态为 "未完成"
```

#### AC2: 空标题无法创建

```gherkin
Given 用户在任务列表页面
When 用户没有输入任何内容
  And 按下回车键或点击添加按钮
Then 不创建新任务
  And 显示提示 "请输入任务内容"
```

#### AC3: 标题长度限制

```gherkin
Given 用户在任务列表页面
When 用户输入超过 200 字符的任务标题
Then 只保留前 200 个字符
  And 显示提示 "任务标题最多 200 字符"
```

### 技术说明

- API: `POST /api/tasks`
- 前端组件: `TaskInput`
- 数据存储: tasks 表

| 状态 | Ready for Dev |

### 故事描述

**作为** 用户，

**我想要** 查看我的所有任务，

**以便** 了解有哪些事情需要完成。

### 验收标准

#### AC1: 显示任务列表

```gherkin
Given 用户有 5 个任务
When 用户访问任务列表页面
Then 显示所有 5 个任务
  And 任务按创建时间倒序排列（最新在前）
  And 每个任务显示标题和状态
```

#### AC2: 空列表提示

```gherkin
Given 用户没有任何任务
When 用户访问任务列表页面
Then 显示友好提示 "暂无任务，快去创建一个吧！"
```

#### AC3: 区分完成状态

```gherkin
Given 用户有已完成和未完成的任务
When 用户查看任务列表
Then 未完成任务显示在上方
  And 已完成任务显示为划线/灰色状态
```

### 技术说明

- API: `GET /api/tasks`
- 前端组件: `TaskList`, `TaskItem`

## US-003: 完成任务

### 基本信息

| 属性 | 内容 |
|------|------|
| 故事 ID | US-003 |
| 标题 | 用户可以标记任务为完成 |
| 优先级 | Must Have |
| 故事点 | 2 |
| 状态 | Ready for Dev |

### 故事描述

**作为** 用户，

**我想要** 标记任务为已完成，

**以便** 追踪我的进度并获得成就感。

### 验收标准

#### AC1: 完成任务

```gherkin
Given 存在一个未完成的任务 "买牛奶"
When 用户点击任务左侧的复选框
Then 任务状态变为 "已完成"
  And 任务标题显示删除线样式
  And 任务移动到已完成区域
```

#### AC2: 取消完成

```gherkin
Given 存在一个已完成的任务 "买牛奶"
When 用户再次点击任务的复选框
Then 任务状态变为 "未完成"
  And 任务恢复正常样式
  And 任务移动到未完成区域
```

### 技术说明

- API: `PUT /api/tasks/:id`
- 状态字段: `completed: boolean`

## US-004: 删除任务

### 基本信息

| 属性 | 内容 |
|------|------|
| 故事 ID | US-004 |
| 标题 | 用户可以删除任务 |
| 优先级 | Must Have |
| 故事点 | 1 |
| 状态 | Ready for Dev |

### 故事描述

**作为** 用户，

**我想要** 删除不再需要的任务，

**以便** 保持任务列表的整洁。

### 验收标准

#### AC1: 删除任务

```gherkin
Given 存在一个任务 "买牛奶"
When 用户点击任务右侧的删除按钮
Then 显示确认对话框 "确定要删除这个任务吗？"
When 用户确认删除
Then 任务从列表中移除
  And 显示提示 "任务已删除"
```

#### AC2: 取消删除

```gherkin
Given 存在一个任务 "买牛奶"
When 用户点击删除按钮
  And 在确认对话框中选择 "取消"
Then 任务保持不变
```

### 技术说明

- API: `DELETE /api/tasks/:id`


## US-006: 任务分类

### 基本信息

| 属性 | 内容 |
|------|------|
| 故事 ID | US-006 |
| 标题 | 用户可以为任务添加分类 |
| 优先级 | Could Have |
| 故事点 | 5 |
| 状态 | Backlog |

### 故事描述

**作为** 用户，

**我想要** 为任务添加分类标签，

**以便** 更好地组织和筛选任务。

### 验收标准

#### AC1: 添加分类

```gherkin
Given 用户在创建或编辑任务
When 用户选择一个分类（如"工作"、"个人"、"购物"）
Then 任务带有该分类标签
```

#### AC2: 按分类筛选

```gherkin
Given 用户有不同分类的任务
When 用户选择 "工作" 分类筛选
Then 只显示 "工作" 分类的任务
```

### 技术说明

- 需要新增 categories 表
- 任务与分类为多对一关系
- MVP 后期或 v1.1 实现
