### 7.2 API 列表

### 7.3 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {
    // 业务数据
  },
  "timestamp": "2026-02-02T10:00:00Z"
}
```

### 7.4 错误码定义

| 错误码 | 描述 | 处理建议 |
|--------|------|----------|
| 200 | 成功 | - |
| 400 | 请求参数错误 | 检查参数 |
| 401 | 未认证 | 登录 |
| 403 | 无权限 | 申请权限 |
| 404 | 资源不存在 | 检查 ID |
| 500 | 服务器错误 | 联系管理员 |

### 8.1 数据模型

#### 核心实体关系图

```
┌─────────────┐       ┌─────────────┐
│    User     │       │    Order    │
├─────────────┤       ├─────────────┤
│ id          │───┐   │ id          │
│ username    │   │   │ user_id     │──┐
│ email       │   └──►│ status      │  │
│ created_at  │       │ total       │  │
└─────────────┘       │ created_at  │  │
                      └─────────────┘  │
                                       │
                      ┌─────────────┐  │
                      │ OrderItem   │  │
                      ├─────────────┤  │
                      │ id          │  │
                      │ order_id    │◄─┘
                      │ product_id  │
                      │ quantity    │
                      │ price       │
                      └─────────────┘
```

### 8.2 核心表设计

#### users 表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| email | VARCHAR(100) | UNIQUE, NOT NULL | 邮箱 |
| password_hash | VARCHAR(255) | NOT NULL | 密码哈希 |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | ON UPDATE NOW() | 更新时间 |

### 8.3 索引设计

| 表 | 索引名 | 字段 | 类型 | 说明 |
|----|--------|------|------|------|
| users | idx_email | email | UNIQUE | 邮箱唯一索引 |
| orders | idx_user_id | user_id | NORMAL | 用户订单查询 |

## 9. 非功能性设计

### 9.1 性能设计

| 指标 | 目标 | 实现方案 |
|------|------|----------|
| 响应时间 | P95 < 200ms | 缓存、异步处理 |
| 吞吐量 | 1000 QPS | 水平扩展、负载均衡 |
| 并发用户 | 10000 | 连接池、限流 |

### 9.2 安全设计

- **认证**：JWT Token
- **授权**：RBAC 角色权限
- **数据安全**：敏感数据加密、HTTPS
- **审计**：操作日志

### 9.3 可靠性设计

- **高可用**：多实例部署、自动故障转移
- **容灾**：数据备份、异地容灾
- **监控**：应用监控、告警机制

### 9.4 可扩展性设计

- **水平扩展**：无状态设计
- **垂直拆分**：按业务域拆分
- **缓存策略**：多级缓存
