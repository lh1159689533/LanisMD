## 验收标准 (Acceptance Criteria)

使用 Given-When-Then 格式：

### 场景 1：[场景名称]

```gherkin
Given [前置条件/初始状态]
  And [额外条件]（可选）
When [用户执行的动作]
  And [额外动作]（可选）
Then [期望的结果/系统响应]
  And [额外验证]（可选）
```

### 场景 2：[场景名称]

```gherkin
Given [前置条件]
When [动作]
Then [结果]
```

### 场景 3：边界条件/异常情况

```gherkin
Given [异常前置条件]
When [用户动作]
Then [系统应该如何处理]
```

## 模板示例

### 示例：用户登录

**故事 ID**: US-001

**作为** 注册用户，

**我想要** 使用邮箱和密码登录系统，

**以便** 访问我的个人账户和数据。

**验收标准**：

```gherkin
# 场景 1：成功登录
Given 我是已注册用户且账户状态正常
  And 我在登录页面
When 我输入正确的邮箱和密码
  And 点击登录按钮
Then 系统应该验证我的凭据
  And 跳转到用户主页
  And 显示欢迎信息

# 场景 2：登录失败 - 密码错误
Given 我在登录页面
When 我输入正确的邮箱但错误的密码
  And 点击登录按钮
Then 系统显示"邮箱或密码错误"提示
  And 保留已输入的邮箱
  And 清空密码输入框

# 场景 3：账户锁定
Given 我已连续 5 次输入错误密码
When 我再次尝试登录
Then 系统显示"账户已锁定，请 30 分钟后重试"
  And 禁用登录按钮 30 分钟
```

**优先级**: Must Have
**故事点**: 3
