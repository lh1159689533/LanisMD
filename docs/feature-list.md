# LanisMD Feature List

> **目的**: 整理 LanisMD 当前尚未支持的功能清单，作为后续迭代规划的依据
> **对标产品**: Typora
> **产品定位**: 面向开发者/程序员的开源免费 Markdown 编辑器
> **创建日期**: 2026-04-12
> **基于版本**: 当前 main 分支

***

## 状态说明

| 标记    | 含义              |
| ----- | --------------- |
| `NEW` | 全新功能，当前无任何实现    |
| `WIP` | 有部分实现/UI 骨架，需补齐 |

## 优先级说明

| 等级 | 含义                             |
| -- | ------------------------------ |
| P0 | 基础体验 -- 不实现则无法作为日常编辑器使用        |
| P1 | 核心竞争力 -- 达到 Typora 80% 体验的关键功能 |
| P2 | 体验打磨 -- 追平 Typora 细节，形成差异化     |
| P3 | 长尾生态 -- 长期演进，建立竞争壁垒            |

***

## P0: 基础体验

> 目标: 让 LanisMD 可作为日常 Markdown 编辑器真正使用

### 大纲面板完善

* **优先级**: P0

* **状态**: Done

* **描述**: Sidebar 已有大纲 Tab，但内容渲染和交互不完整，长文档导航依赖此功能。

* **验收标准**:

  * [x] 实时解析文档标题生成层级目录树

  * [x] 点击标题项跳转到对应位置

  * [x] 当前光标所在标题高亮显示

  * [x] 滚动文档时大纲面板自动同步高亮

  * [x] `Cmd+Shift+L` 快捷键切换大纲面板

* **涉及模块**: Sidebar.tsx、ProseMirror heading 解析、滚动同步逻辑

* **预估工作量**: 中

### 源码/预览滚动位置同步

* **优先级**: P0

* **状态**: NEW

* **描述**: 在 WYSIWYG 和源码模式间切换时，保持滚动位置一致。

* **验收标准**:

  * [ ] 从 WYSIWYG 切换到源码模式时，光标/视口位置对应

  * [ ] 从源码模式切换回来时同样保持位置

* **涉及模块**: 编辑器模式切换逻辑、ProseMirror/CodeMirror 位置映射

* **预估工作量**: 中

### 智能标点配对

* **优先级**: P0

* **状态**: NEW

* **描述**: 输入左括号自动补全右括号，光标置于中间。

* **验收标准**:

  * [ ] 自动配对 `()` `[]` `{}` `""` `''` ` `` `

  * [ ] 选中文字后输入配对符号自动包裹

  * [ ] 考虑对链接语法的影响：用户输入`[text](url)`时，正确渲染

  * [ ] 考虑对图片语法的影响：用户输入`![alt](url)`时，正确渲染

* **涉及模块**: ProseMirror inputRules / keymap

* **预估工作量**: 小

### 高亮语法（==text==）

* **优先级**: P0

* **状态**: Done

* **描述**: `==highlighted text==` 语法，技术文档中常用于标注重点内容。

* **验收标准**:

  * [ ] 输入 `==text==` 自动渲染为高亮样式

  * [ ] 高亮颜色跟随主题变化

  * [ ] Tooltip 工具栏提供高亮按钮

  * [ ] 导出时保留高亮样式

* **涉及模块**: 自定义 mark 插件（类似 underline-mark.ts）

* **预估工作量**: 小

### 搜索集成（类似vscode）

* **优先级**: P0

* **状态**: Done

* **描述**: 搜索 UI 组件（`SearchReplace.tsx`）已实现，但未集成到编辑器中。

* **验收标准**:

  * [ ] `Cmd+Shift+F` 打开搜索面板（需区分操作系统）

  * [ ] 搜索结果在编辑器中实时高亮（ProseMirror decoration）

  * [ ] 支持上一个/下一个导航跳转

  * [ ] 支持限定范围：包含的文件/排除的文件

* **涉及模块**: SearchReplace.tsx 集成、ui-store（面板状态）、useShortcuts、ProseMirror 搜索装饰

* **预估工作量**: 中

### 文件快速打开（Cmd+P）

* **优先级**: P0

* **状态**: DONE

* **描述**: 开发者习惯的快速文件切换方式（VS Code 同款）。在当前打开的文件夹中模糊搜索文件名。

* **验收标准**:

  * [x] `Cmd+P` 弹出搜索框

  * [x] 输入关键字模糊匹配当前文件夹中的 .md 文件

  * [x] 结果实时过滤，支持键盘上下选择和回车打开

  * [x] 显示文件名 + 相对路径

  * [x] 最近打开的文件排在前面

* **涉及模块**: 新增 QuickOpen 组件、文件索引服务、模糊搜索算法

* **预估工作量**: 中

### 数学公式支持（LaTeX）

* **优先级**: P0

* **状态**: DONE

* **描述**: 技术文档的高频需求，支持行内公式 `$...$` 和块级公式 `$$...$$`。

* **验收标准**:

  * [ ] 行内公式 `$E=mc^2$` 实时渲染

  * [ ] 块级公式 `$$...$$` 独立段落渲染

  * [ ] WYSIWYG 模式点击公式进入编辑态，失焦后渲染

  * [ ] 源码模式正常显示 LaTeX 源码

  * [ ] 导出 HTML/PDF 时正确渲染公式

* **涉及模块**: editor-setup（KaTeX/MathJax 集成）、Milkdown math 插件

* **预估工作量**: 中

### 快捷键体系完善

* **优先级**: P0

* **状态**: DONE

* **描述**: 基础快捷键已有（保存/打开/新建/侧边栏），但编辑类快捷键覆盖不全。

* **验收标准**:

  * [ ] `Cmd+1` ~ `Cmd+6` 切换标题级别

  * [ ] `Cmd+K` 插入/编辑链接

  * [ ] `Cmd+F` 搜索、`Cmd+H` 替换

  * [ ] `Cmd+/` 切换源码模式

  * [ ] `Cmd+Shift+L` 切换大纲面板

  * [ ] `Cmd+Shift+P` 打开命令面板

  * [ ] 快捷键设置面板可查看和自定义

* **涉及模块**: useShortcuts.ts、ProseMirror keymap

* **预估工作量**: 中

### YAML Front Matter 支持

* **优先级**: P0

* **状态**: DONE

* **描述**: 开发者写博客/文档几乎必须。Hugo/Jekyll/Hexo/VitePress 等静态站点生成器均依赖 Front Matter。

* **验收标准**:

  * [ ] 正确解析文档头部的 `---` YAML 块

  * [ ] WYSIWYG 模式下以特殊区域展示（可折叠/展开，区别于正文）

  * [ ] 不破坏 Front Matter 内容的序列化/反序列化

  * [ ] 源码模式下正常显示和编辑

* **涉及模块**: 编辑器插件（remark-frontmatter 生态）、editor-setup

* **预估工作量**: 小

### 代码块折叠

* **优先级**: P0

* **状态**: DONE

* **描述**: 长代码块可收起，提高文档可读性。

* **验收标准**:

  * [ ] 代码块左侧显示折叠/展开按钮

  * [ ] 折叠后显示语言标签和首行预览

  * [ ] 折叠状态不影响文档内容

* **预估工作量**: 小

### 跨平台标题栏完善

* **优先级**: P0

* **状态**: DONE

* **描述**: Windows/Linux 平台缺少自定义窗口控制按钮。

* **验收标准**:

  * [ ] Windows/Linux 渲染最小化/最大化/关闭按钮

  * [ ] 双击标题栏触发最大化/还原

  * [ ] 按钮风格与平台原生风格一致

* **涉及模块**: TitleBar.tsx、平台检测

* **预估工作量**: 小

### \[TOC] 目录生成

* **优先级**: P0

* **状态**: DONE

* **描述**: 在文档中插入 `[TOC]` 自动生成可点击的目录列表，随标题变化实时更新。

* **验收标准**:

  * [ ] 输入 `[TOC]` 自动渲染为目录列表

  * [ ] 目录内容根据文档标题实时更新

  * [ ] 点击目录项跳转到对应标题

  * [ ] 导出时保留目录结构

* **预估工作量**: 小

### 上标/下标语法

* **优先级**: P0

* **描述**: `^上标^` 和 `~下标~` 语法支持。

## P1: 核心竞争力

> 目标: 达到 Typora 80% 核心体验，具备"替代品"资格

### F-P1-001: 导出 PDF

* **优先级**: P1

* **状态**: NEW

* **描述**: 最高频的导出需求。当前无可用的导出功能。

* **验收标准**:

  * [ ] 支持将当前文档导出为 PDF 文件

  * [ ] 导出结果保留主题样式（字体、颜色、间距）

  * [ ] 正确渲染表格、代码块（含语法高亮）、图片、Mermaid 图表、数学公式

  * [ ] 支持页边距、纸张大小设置

  * [ ] `Cmd+P` 触发导出

* **涉及模块**: Tauri webview 打印 API / wkhtmltopdf、导出服务

* **预估工作量**: 大

### F-P1-002: 导出 HTML

* **优先级**: P1

* **状态**: NEW

* **描述**: 第二高频导出，也是 PDF 导出的基础能力。

* **验收标准**:

  * [ ] 导出完整的独立 HTML 文件（内联 CSS）

  * [ ] 保留当前主题样式

  * [ ] 正确渲染所有 Markdown 元素

  * [ ] 可选择是否内联图片（Base64）或保持外部引用

* **涉及模块**: Markdown 解析器（remark/rehype）、HTML 模板、主题内联

* **预估工作量**: 中

### F-P1-003: 打字机/专注模式

* **优先级**: P1

* **状态**: NEW

* **描述**: Typora 标志性写作体验。打字机模式让当前行始终保持在屏幕中央，专注模式淡化非当前段落。

* **验收标准**:

  * [ ] 打字机模式: 编辑时当前行自动滚动到编辑区垂直中央

  * [ ] 专注模式: 非当前段落文字降低透明度（如 30% opacity）

  * [ ] 两种模式可独立或组合开启

  * [ ] 设置面板和状态栏提供开关

  * [ ] 模式切换有平滑过渡动画

* **涉及模块**: ProseMirror 滚动控制、CSS 段落淡化、editor-store 状态

* **预估工作量**: 中

### F-P1-008: 编辑器宽度调节

* **优先级**: P1

* **状态**: NEW

* **描述**: 不同屏幕尺寸和内容类型需要不同的编辑区宽度。

* **验收标准**:

  * [ ] 设置中可调节编辑区最大宽度（如 720px / 900px / 100%）

  * [ ] 状态栏或设置面板提供快捷切换

  * [ ] 宽度切换有平滑过渡

* **涉及模块**: CSS 变量调整、settings-store

* **预估工作量**: 小

## P2: 体验打磨

> 目标: 细节追平 Typora，强化差异化优势

### F-P2-001: 导出 Word (docx)

* **优先级**: P2

* **状态**: NEW

* **描述**: 与非技术人员协作分享文档时需要。

* **验收标准**:

  * [ ] 导出为 .docx 格式，保留基本排版

  * [ ] 正确渲染表格、代码块、图片

  * [ ] 可选是否嵌入图片

* **涉及模块**: Pandoc 集成或 docx 生成库

* **预估工作量**: 大

### F-P2-002: 脚注支持

* **优先级**: P2

* **状态**: NEW

* **描述**: 学术文档和技术规范中常用的引用注释功能。

* **验收标准**:

  * [ ] 支持 `[^1]` 引用和 `[^1]: 注释内容` 定义语法

* **优先级**: P2

* **状态**: NEW

## P3: 长尾功能与生态

> 目标: 建立差异化竞争优势，超越 Typora

### F-P3-001: 自动更新

* **优先级**: P3

* **状态**: NEW

* **描述**: 桌面应用必备的版本更新机制。

* **验收标准**:

  * [ ] 启动时检查更新

  * [ ] 提示新版本并支持一键下载安装
