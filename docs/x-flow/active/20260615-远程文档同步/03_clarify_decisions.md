# 需求澄清决策记录

> Phase 03-05 (CLARIFY) 产出 | 2026-06-15

## 开放问题决策

### 1. 大文件处理
- **决策**: 初版不支持 Git LFS
- **行为**: 超过 25MB 的文件在上传时跳过，并给用户明确提示
- **原因**: GitHub REST API 单次上传限制为 25MB（Base64 编码后约 33MB 请求体）

### 2. 远程目录浏览
- **决策**: 初版不支持分页
- **行为**: 单次最多加载 1000 个文件，超出时提示用户缩小目标路径范围
- **原因**: 初版简化实现，满足绝大多数用户场景

### 3. 文件过滤策略
- **决策**: 不解析 .gitignore，改为配置项中增加白名单/黑名单规则
- **行为**: 用户在远程仓库配置中可设置文件过滤规则（glob 模式），上传时按规则过滤
- **配置项新增**:
  - `includePatterns`（白名单，默认 `["*.md", "*.png", "*.jpg", "*.jpeg", "*.gif", "*.svg", "*.webp"]`）
  - `excludePatterns`（黑名单，默认 `[]`）
- **优先级**: 黑名单优先于白名单

### 4. 拉取与缓存策略（重要变更）
- **决策**: 无需缓存清理策略，改为"本地文件夹 ↔ 远程仓库"双向增量同步模式
- **核心流程变更**:
  1. 拉取时用户指定一个**本地文件夹**作为同步目标
  2. 对比本地文件夹与远程文件的 **MD5**，只拉取 MD5 不同的文件（增量拉取）
  3. 拉取后所有操作（增删改查）都基于**本地文件夹**进行
  4. 用户可手动触发将本地文件夹**同步回远程**（增量上传）
- **影响**: PRD §2.3 描述需要调整，不再有"临时/缓存目录"概念

## 对 PRD 的修订摘要

1. §2.1 配置项新增: `includePatterns`（白名单）、`excludePatterns`（黑名单）
2. §2.3 工作流调整: 拉取到用户指定本地文件夹，非临时缓存目录；基于 MD5 增量对比
3. §4 约束新增: 单文件 >25MB 跳过；远程目录最多 1000 文件
4. §6 开放问题全部关闭
