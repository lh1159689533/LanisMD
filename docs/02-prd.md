## 1. 产品概述

### 1.1 产品愿景

> 打造一款**开源、轻量、极致流畅**的所见即所得（WYSIWYG）Markdown 桌面编辑器，成为 Typora 最优秀的开源替代品。

### 1.2 产品使命

- 为 Markdown 写作者提供一个**无干扰、沉浸式**的写作环境
- 以 **Tauri 轻量架构**证明桌面应用不必依赖 Electron 的臃肿
- 以**完全开源**的姿态填补开源 WYSIWYG Markdown 编辑器的市场空白

### 1.3 核心价值主张

> **「比 Typora 更开放，比 MarkText 更优秀，比 Electron 更轻量」**

| 价值维度 | 具体体现 |
|----------|----------|
| **更开放** | 完全开源，用户可自由定制、扩展、审计 |
| **更优秀** | 多标签页、命令面板等现代编辑器特性，超越 Typora 的单窗口体验 |
| **更轻量** | 基于 Tauri，安装包约 3-5MB（vs Electron ~70-100MB），内存占用降低 50%+ |

### 1.4 目标用户

直接引用 brainstorm 阶段的三个核心用户画像（详见 [01-brainstorm.md §2](./01-brainstorm.md#2-目标用户画像)）：

#### 画像 A：独立开发者 / 技术博主「阿文」

- **核心诉求**：打开即写、所见即所得、代码高亮、数学公式、Mermaid 图表、轻量快速
- **典型场景**：撰写技术博客、项目 README、学习笔记、GitHub Issue

#### 画像 B：研究生 / 学术写作者「小林」

- **核心诉求**：免费开源、数学公式渲染、表格编辑、PDF 导出、跨平台一致
- **典型场景**：整理课程笔记、撰写实验报告、毕业论文草稿、读书笔记

#### 画像 C：项目经理 / 知识整理者「老陈」

- **核心诉求**：多标签页（刚需）、快捷键驱动、大纲视图、搜索替换、启动快
- **典型场景**：编写技术方案、会议纪要、需求文档、临时笔记

#### 核心共性需求

**所见即所得 + 轻量快速 + 免费开源**——这是 MyTypora 必须满足的三重底线。

### 1.5 成功指标

作为个人学习项目，成功定义为以下可衡量的标准：

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **核心编辑可用性** | MVP 可用于日常写作 | 能打开、编辑、保存一篇包含标题/列表/代码块/图片/表格的 Markdown 文档 |
| **启动速度** | < 2 秒（冷启动） | 从点击图标到可编辑状态的时间 |
| **安装包体积** | < 10MB | Tauri 打包后的安装程序大小 |
| **GitHub Stars** | 100+（发布 3 个月内） | 衡量社区关注度 |
| **个人成长** | 掌握 Tauri + Milkdown 全栈开发 | 技术学习目标达成 |

## 2. 功能需求清单

### 2.1 P0 - 必须有（MVP 核心）

> 定义：没有这些功能，产品无法称为"Markdown 编辑器"。MVP 阶段必须交付。

#### F-001：WYSIWYG Markdown 编辑

**功能描述**：基于 Milkdown 7.x 实现所见即所得的 Markdown 编辑体验。编辑时显示 Markdown 标记，光标离开后自动隐藏标记，只显示渲染后的格式化内容。

**验收标准**：

- AC1：支持 h1-h6 六级标题，通过 `#` 标记或快捷键（Ctrl/Cmd+1~6）创建
- AC2：支持粗体（`**text**` / Ctrl/Cmd+B）、斜体（`*text*` / Ctrl/Cmd+I）、删除线（`~~text~~`）
- AC3：支持引用块（`>`），支持多级嵌套引用（`>>`）
- AC4：支持分割线（`---` / `***`），渲染为水平线
- AC5：光标所在的段落保持可编辑态（显示标记），非焦点段落呈现渲染态（隐藏标记）
- AC6：格式切换无感知延迟，键盘输入到屏幕渲染 < 50ms

**优先级**：P0

---

#### F-002：列表编辑

**功能描述**：支持有序列表、无序列表和任务列表（GFM）的创建和编辑。

**验收标准**：

- AC1：支持无序列表（`-` / `*` / `+`），支持 4 级嵌套
- AC2：支持有序列表（`1.` / `2.` / ...），自动递增编号
- AC3：支持任务列表（`- [ ]` / `- [x]`），可点击切换完成状态
- AC4：支持列表项的缩进（Tab）和反缩进（Shift+Tab）
- AC5：列表项之间可通过 Enter 创建新项，空行 Enter 退出列表
- AC6：在列表顶部空行按 Backspace 可退回普通段落

**优先级**：P0

---

#### F-003：代码块

**功能描述**：支持围栏代码块的创建和编辑，包含语法高亮和语言选择。

**验收标准**：

- AC1：支持 `` ``` `` 围栏代码块，支持指定语言标识（`` ```js `` 等）
- AC2：代码块支持语法高亮，至少支持以下语言：JavaScript/TypeScript、Python、Go、Rust、Java、HTML、CSS、JSON、Bash、SQL
- AC3：代码块右上角显示语言选择下拉菜单，可切换代码语言
- AC4：代码块内的文本不会被 Markdown 格式化解析
- AC5：代码块支持 Tab 缩进（插入 2 空格或 1 Tab）
- AC6：代码块内容可正常复制/粘贴，保持缩进格式

**优先级**：P0

---

#### F-004：图片插入

**功能描述**：支持通过拖拽、粘贴、路径输入三种方式插入图片。

**验收标准**：

- AC1：支持将本地图片文件**拖拽**到编辑器中插入，生成相对路径 `![](./image.png)`
- AC2：支持在编辑器中直接**粘贴**剪贴板中的图片，自动保存到当前文档所在目录（或用户配置的图片目录），并插入引用
- AC3：支持通过 Markdown 语法手动输入图片路径 `![alt](path)`
- AC4：支持本地绝对路径和相对路径两种引用方式
- AC5：若插入的网络图片 URL 无法加载，显示 alt 文本或占位符，不阻塞编辑
- AC6：粘贴图片时自动生成唯一文件名（如 `image-20260327-143022.png`），避免文件名冲突

**优先级**：P0

---

#### F-005：链接插入与编辑

**功能描述**：支持超链接的创建、编辑和删除。

**验收标准**：

- AC1：支持 `[text](url)` 语法插入超链接
- AC2：支持 Ctrl/Cmd+K 快捷键快速插入/编辑链接
- AC3：支持粘贴 URL 时自动识别并提示转换为链接
- AC4：点击已渲染的链接可在默认浏览器中打开
- AC5：链接文本和 URL 可独立编辑
- AC6：支持空链接文本时显示 URL 本身

**优先级**：P0

---

#### F-006：表格创建与编辑

**功能描述**：支持 GFM 表格的创建和基本编辑操作。

**验收标准**：

- AC1：支持通过快捷键或菜单创建指定行列数的表格（默认 3×3）
- AC2：支持在表格中通过 Tab 键在单元格间移动
- AC3：支持通过菜单或快捷键在表格中插入/删除行
- AC4：支持通过菜单或快捷键在表格中插入/删除列
- AC5：支持表格表头行（第一行）与非表头行的区分渲染
- AC6：表格支持对齐方式：左对齐（默认）、居中对齐、右对齐

**优先级**：P0

---

#### F-007：文件操作

**功能描述**：支持 Markdown 文件的新建、打开、保存和另存为。

**验收标准**：

- AC1：支持新建空白 Markdown 文件（.md），自动命名为"未命名.md"
- AC2：支持通过系统原生文件对话框打开一个或多个 .md 文件
- AC3：支持 Ctrl/Cmd+S 保存文件；若为新文件，触发"另存为"对话框
- AC4：支持 Ctrl/Cmd+Shift+S 另存为，可选择保存路径和文件名
- AC5：保存时确保文件编码为 UTF-8，换行符为 LF
- AC6：打开非 .md 文件（如 .txt）时给出提示，用户确认后以纯文本模式打开
- AC7：保存前如有未保存修改，标签页标题显示修改标记（如 `*` 或圆点）

**优先级**：P0

---

#### F-008：多标签页管理

**功能描述**：支持同时打开多个文件并在标签页间切换，这是对标 Typora 的核心差异化功能。

**验收标准**：

- AC1：支持通过标签栏显示已打开的文件列表
- AC2：支持点击标签切换到对应文件，切换时保留各文件的编辑状态（光标位置、滚动位置）
- AC3：支持通过标签上的关闭按钮或 Ctrl/Cmd+W 关闭当前标签
- AC4：支持中键点击标签关闭对应标签
- AC5：关闭非最后一个标签时，自动切换到相邻标签
- AC6：关闭最后一个标签时，自动新建空白标签页（应用不退出）
- AC7：每个标签页独立显示文件名，有未保存修改的标签显示修改标记
- AC8：支持鼠标拖拽标签页调整排列顺序

**优先级**：P0

---

#### F-009：最近打开文件记录

**功能描述**：记录用户最近打开的文件列表，方便快速恢复工作。

**验收标准**：

- AC1：在欢迎页面或菜单中展示最近打开的文件列表，最多记录 20 条
- AC2：点击最近文件列表中的条目可直接打开对应文件
- AC3：列表按最近打开时间倒序排列
- AC4：若文件已被删除或移动，点击后给出提示而非崩溃
- AC5：最近文件列表在应用重启后仍然保留（持久化存储）

**优先级**：P0

---

#### F-010：自动保存

**功能描述**：编辑器自动保存当前文档的修改，防止数据丢失。

**验收标准**：

- AC1：对已保存过的文件，每 5 秒自动保存一次（静默保存，无弹窗）
- AC2：对未保存的新文件（"未命名.md"），不执行自动保存
- AC3：自动保存时使用原子写入（先写入临时文件，再重命名），防止写入中断导致文件损坏
- AC4：自动保存失败时（如磁盘空间不足、权限问题），在状态栏显示警告图标
- AC5：应用非正常退出（如崩溃、强制关闭）后，下次启动时可恢复最后一次自动保存的内容
- AC6：用户可在设置中关闭自动保存功能

**优先级**：P0

---

#### F-011：撤销 / 重做

**功能描述**：支持编辑操作的撤销和重做。

**验收标准**：

- AC1：支持 Ctrl/Cmd+Z 撤销，Ctrl/Cmd+Shift+Z 或 Ctrl/Cmd+Y 重做
- AC2：撤销/重做粒度为"一次有意义操作"（如输入一个字符、粘贴一段文本、插入一个图片），而非逐字符撤销
- AC3：保存文件不中断撤销历史（保存后仍可撤销到保存前的状态）
- AC4：撤销/重做栈深度至少支持 200 步
- AC5：菜单栏显示当前可撤销/重做的操作描述（如"撤销：输入文字"）

**优先级**：P0

---

### 2.2 P1 - 应该有

> 定义：对用户体验有显著提升的功能，在 MVP 基本可用后优先实现。

#### F-012：PDF 导出

**功能描述**：将当前 Markdown 文档导出为 PDF 文件。

**验收标准**：

- AC1：通过菜单或快捷键（Ctrl/Cmd+P）触发导出 PDF
- AC2：导出的 PDF 保留文档的格式化样式（标题层级、粗体/斜体、代码块、表格、图片）
- AC3：导出前可选择保存路径和文件名
- AC4：导出过程中显示进度提示，完成后显示成功通知
- AC5：包含中文内容时正确渲染，无乱码或缺失字体
- AC6：代码块在 PDF 中保持语法高亮颜色

**优先级**：P1

---

#### F-013：HTML 导出

**功能描述**：将当前 Markdown 文档导出为独立的 HTML 文件。

**验收标准**：

- AC1：通过菜单触发导出 HTML
- AC2：导出的 HTML 为单文件（内嵌 CSS 和图片 base64 编码），可直接在浏览器中打开
- AC3：保留文档的格式化样式
- AC4：代码块在导出的 HTML 中保持语法高亮
- AC5：数学公式（如已渲染）在 HTML 中正确显示

**优先级**：P1

---

#### F-014：KaTeX 数学公式渲染

**功能描述**：支持 LaTeX 数学公式的输入和实时渲染。

**验收标准**：

- AC1：支持行内公式 `$E=mc^2$`，渲染为行内数学公式
- AC2：支持块级公式 `$$E=mc^2$$`，渲染为居中显示的数学公式
- AC3：公式编辑时光标所在公式显示 LaTeX 源码，非焦点公式显示渲染结果
- AC4：公式渲染使用 KaTeX，支持常见 LaTeX 语法（分数、根号、矩阵、希腊字母、上下标等）
- AC5：包含公式的文档导出 PDF/HTML 时公式正确渲染

**优先级**：P1

---

#### F-015：Mermaid 图表渲染

**功能描述**：支持 Mermaid 语法代码块的实时渲染为图表。

**验收标准**：

- AC1：检测 `` ```mermaid `` 代码块，渲染为对应的流程图/时序图/甘特图等
- AC2：支持 Mermaid 常见图表类型：flowchart、sequence diagram、class diagram、gantt、pie chart
- AC3：图表编辑时光标所在代码块显示 Mermaid 源码，非焦点时显示渲染后的图表
- AC4：Mermaid 语法错误时显示错误提示，不阻塞编辑
- AC5：Mermaid.js 采用懒加载策略，仅在文档包含 Mermaid 代码块时加载，避免影响启动速度
- AC6：渲染结果支持缩放查看

**优先级**：P1

---

#### F-016：自动目录（TOC）

**功能描述**：根据文档标题层级自动生成目录。

**验收标准**：

- AC1：支持通过 `[toc]` 或 `[[toc]]` 语法在文档中插入自动目录
- AC2：目录根据文档中的 h1-h6 标题自动生成，层级对应
- AC3：目录项可点击，跳转到对应标题位置
- AC4：文档标题发生变化时，目录自动更新
- AC5：嵌套层级通过缩进清晰展示

**优先级**：P1

---

#### F-017：搜索与替换

**功能描述**：支持在当前文档中搜索和替换文本内容。

**验收标准**：

- AC1：支持 Ctrl/Cmd+F 打开搜索栏，输入关键词后实时高亮所有匹配项
- AC2：支持 Ctrl/Cmd+H 打开搜索与替换栏
- AC3：支持大小写敏感/不敏感切换
- AC4：支持正则表达式搜索
- AC5：支持"下一个"/"上一个"逐个定位匹配项
- AC6：支持"全部替换"功能，替换前显示匹配数量确认
- AC7：搜索栏支持 Esc 关闭
- AC8：显示当前匹配项索引和总匹配数量（如 "3/15"）

**优先级**：P1

---

#### F-018：大纲视图

**功能描述**：提供文档标题结构的树状视图，支持快速导航。

**验收标准**：

- AC1：侧边栏提供"大纲"面板，列出文档中所有标题（h1-h6）
- AC2：标题按层级缩进展示，形成树状结构
- AC3：点击大纲中的标题项，编辑器光标跳转到对应位置
- AC4：大纲随文档内容实时更新（新增/删除标题时自动刷新）
- AC5：支持 Ctrl/Cmd+Shift+L 快捷键切换大纲视图的显示/隐藏
- AC6：当前光标所在标题在大纲中高亮显示

**优先级**：P1

---

#### F-019：源码模式切换

**功能描述**：支持在 WYSIWYG 模式和 Markdown 源码模式之间一键切换。

**验收标准**：

- AC1：支持 Ctrl/Cmd+/ 快捷键切换源码模式
- AC2：源码模式下显示原始 Markdown 文本，支持编辑
- AC3：从源码模式切换回 WYSIWYG 模式时，正确解析 Markdown 为渲染视图
- AC4：两种模式之间切换时光标位置尽可能保持一致
- AC5：切换过程中不丢失任何内容或格式

**优先级**：P1

---

#### F-020：字数 / 行数统计

**功能描述**：在状态栏实时显示文档的字数、字符数、行数等统计信息。

**验收标准**：

- AC1：状态栏显示当前文档的字数（中文按字计、英文按词计）
- AC2：状态栏显示字符数（不含空格）和字符数（含空格）
- AC3：状态栏显示当前文档的总行数
- AC4：状态栏显示预估阅读时间（基于中文 300 字/分钟，英文 200 词/分钟）
- AC5：统计信息随编辑实时更新（无需手动刷新）
- AC6：选中文本时，状态栏显示选中部分的字数统计

**优先级**：P1

---

#### F-021：快捷键自定义

**功能描述**：允许用户自定义编辑器的快捷键绑定。

**验收标准**：

- AC1：设置面板中提供"快捷键"设置页，列出所有可自定义的快捷键
- AC2：用户可点击快捷键条目后按下新的组合键来修改绑定
- AC3：若新快捷键与已有绑定冲突，显示冲突提示，用户确认后覆盖
- AC4：提供"恢复默认"按钮，一键重置所有快捷键为默认值
- AC5：自定义的快捷键绑定持久化保存，重启后生效

**优先级**：P1

---

#### F-022：主题切换（亮色 / 暗色 / 跟随系统）

**功能描述**：支持编辑器的亮色/暗色主题切换，以及跟随系统主题。

**验收标准**：

- AC1：支持亮色主题（默认），提供至少 1 款内置亮色主题

## 4. 非功能需求

### 4.1 性能

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| **冷启动时间** | < 2 秒 | 从点击应用图标到编辑器可接收输入 |
| **热启动时间** | < 1 秒 | 应用已在后台时的恢复时间 |
| **大文档编辑** | 10,000 行文档无明显卡顿 | 打开 10,000 行 .md 文件，滚动和编辑流畅 |
| **大文档打开** | < 3 秒 | 打开 10,000 行 .md 文件到可编辑状态 |
| **内存占用** | < 150MB | 正常编辑一个中等文档（500 行）时的内存使用 |
| **安装包体积** | < 10MB | 各平台安装程序文件大小 |
| **输入延迟** | < 50ms | 从按键到屏幕渲染的延迟 |

### 4.2 兼容性

| 平台 | 最低版本要求 | 说明 |
|------|-------------|------|
| **macOS** | macOS 12 Monterey (x86_64 + Apple Silicon) | Tauri 2.x 要求 |
| **Windows** | Windows 10 (1809+) | WebView2 已预装 |
| **Linux** | 主流发行版（Ubuntu 20.04+、Fedora 36+、Arch） | 需要安装 WebKit2GTK |

- Markdown 语法兼容性：CommonMark 规范 + GFM（GitHub Flavored Markdown）
- 文件兼容性：UTF-8 / UTF-8 BOM / GBK / GB2312 编码识别

### 4.3 安全性

- **无网络通信**：应用不进行任何网络请求（P0-P3 阶段），所有数据本地处理
- **无遥测数据**：不收集用户行为数据
- **本地数据存储**：所有文件和配置存储在用户本地文件系统
- **文件访问权限**：遵循操作系统权限模型，通过 Tauri 权限沙箱访问文件
- **原子写入**：保存文件时使用"写入临时文件 → 重命名"策略，防止数据损坏

### 4.4 可访问性

- **键盘完全可操作**：所有核心功能均可通过键盘完成，不依赖鼠标
- **焦点管理**：清晰的焦点指示，Tab 键可在各 UI 区域间切换
- **高对比度**：默认主题在亮色/暗色模式下均满足 WCAG AA 对比度标准
- **无障碍基础**：为关键 UI 元素提供 ARIA 标签（后续可增强屏幕阅读器支持）

### 4.5 可维护性

- **模块化架构**：前端组件化，Tauri 后端功能按插件划分
- **TypeScript 全量**：前端代码 100% TypeScript，编译时类型检查
- **文档完整**：代码注释 + 架构文档 + API 文档
- **CI/CD**：自动化构建和测试流水线

## 5. 明确不做

> 以下功能在可预见的未来（至少至 P2 阶段）**不会纳入开发计划**，以保持项目聚焦。

| 不做的功能 | 原因 | 备注 |
|-----------|------|------|
| **云端同步** | 定位为纯本地编辑器，避免引入网络依赖和账户系统 | 用户可通过 Git/网盘自行同步文件 |
| **实时协作编辑** | 架构复杂度高，需要后端服务支撑 | 与"本地优先"定位冲突 |
| **移动端（iOS / Android）** | Tauri 移动端尚不成熟，且非核心用户需求 | 聚焦桌面端体验 |
| **插件系统（MVP）** | 增加架构复杂度，MVP 阶段应聚焦核心功能 | 列为 P3 未来探索方向 |
| **数据库存储** | Markdown 文件即存储，无需引入数据库 | 使用文件系统直接管理 |
| **Vim/Emacs 模式** | 增加维护成本，用户群太小 | 非核心需求 |
| **内部链接/双向链接** | 这是知识管理工具的功能，非编辑器核心 | Obsidian 已做得很好 |
| **图片上传到图床** | 需要网络通信和第三方服务集成 | 违背"无网络通信"原则 |
| **Pandoc 导出（Word/Epub/LaTeX）** | 需要安装 Pandoc 外部依赖，增加部署复杂度 | MVP 阶段通过 PDF/HTML 覆盖基本需求 |
| **拼写检查** | 非核心需求，且多语言拼写检查实现复杂 | 可作为社区插件方向 |

## 6. 里程碑规划

### Phase 1：MVP 核心（P0）

> **目标**：实现"打开 → 编辑 → 保存"的核心工作流，验证 Milkdown 编辑体验。

| 里程碑 | 核心交付物 | 预计周期 |
|--------|-----------|---------|
| **M1.1 - 编辑器内核** | Milkdown 集成、WYSIWYG 基础格式、列表、代码块、图片、链接、表格 | 2-3 周 |
| **M1.2 - 文件操作** | 新建/打开/保存/另存为、自动保存、撤销/重做 | 1-2 周 |
| **M1.3 - 标签页** | 多标签页管理、标签切换、拖拽排序、最近文件 | 2 周 |
| **M1.4 - MVP 收尾** | 自定义标题栏、基本菜单、欢迎页、bug 修复 | 1 周 |

**MVP 交付标准**：可以作为一个日常 Markdown 编辑器使用，满足基础写作需求。

### Phase 2：体验增强（P1）

> **目标**：补全重要功能，使编辑器从"能用"升级为"好用"。

| 里程碑 | 核心交付物 | 预计周期 |
|--------|-----------|---------|
| **M2.1 - 导出与渲染** | PDF 导出、HTML 导出、KaTeX 公式、Mermaid 图表 | 2-3 周 |
| **M2.2 - 编辑增强** | 搜索替换、大纲视图、源码模式、字数统计 | 2 周 |
| **M2.3 - 个性化** | 主题切换、快捷键自定义、设置面板 | 2 周 |

### Phase 3：锦上添花（P2）

> **目标**：提升编辑体验的"惊喜感"，探索特色功能。

| 里程碑 | 核心交付物 | 预计周期 |
|--------|-----------|---------|
| **M3.1 - 写作增强** | 打字机模式、专注模式、命令面板 | 1-2 周 |
| **M3.2 - 文件增强** | 文件监听、编码选择、块级拖拽、图片调整 | 2 周 |

### Phase 4：未来探索（P3）

> **目标**：探索前沿功能，保持项目活力。

| 里程碑 | 核心交付物 | 预计周期 |
|--------|-----------|---------|
| **M4.1 - AI 集成** | Ollama 本地 LLM 集成、续写/纠错/摘要 | 2-3 周 |
| **M4.2 - 高级功能** | 分屏编辑、插件系统架构 | 3-4 周 |
