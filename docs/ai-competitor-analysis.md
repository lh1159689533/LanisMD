# LanisMD AI 功能竞品对标分析

> 调研日期：2026-04-23
> 对标对象：Typora (及其 AI 插件生态)、Obsidian (AI 插件生态)、Notion AI
> 目的：验证 LanisMD AI 集成方案是否有遗漏或可借鉴之处

***

## 1. 竞品 AI 能力全景

### 1.1 Typora

Typora **官方不内置 AI 功能**，AI 能力完全来自第三方插件。主流有三个：

| 插件                          | 定位                | 核心能力                                       |
| --------------------------- | ----------------- | ------------------------------------------ |
| **typora-ai-edit** (v3.1.0) | 全能 AI 工作室         | 文本润色/翻译、AI 问答、图表生成、图像分析、网络搜索、飞书集成          |
| **typora-copilot**          | GitHub Copilot 集成 | 类 IDE 的幽灵文本补全（ghost text），光标处实时建议          |
| **typora-plugin-ai**        | 轻量 AI 扩展          | 基于 obgnail/typora\_plugin 的 AI 子模块，基础问答与润色 |

**typora-ai-edit 的关键设计（最成熟）：**

* **触发方式**：`Cmd+E` 快捷键 + 右键菜单（根据上下文动态变化）

* **上下文感知优化**：不仅能处理选中文本，还能结合**整篇文档上下文**进行优化

* **累积式对话**：问答支持追问，保留完整对话历史

* **图表迭代**：生成 Mermaid/SVG 后可通过追问修改（"添加一个审核步骤"）

* **图像分析**：右键图片可让 AI 描述内容、提取数据

* **网络搜索**：集成 Tavily 搜索引擎，生成内容带来源引用

* **自定义 System Prompt**：用户可从根本上定制 AI 行为

* **多模型切换**：GPT/Claude/DeepSeek/Qwen/Kimi，右键菜单快速切换

* **自动检测模型能力**：自动验证模型的搜索/视觉能力，菜单自适应显隐

### 1.2 Obsidian

Obsidian 同样**官方不内置 AI**，但拥有极其丰富的插件生态。三大核心插件形成互补：

| 插件                    | 角色   | 交互方式    | 核心场景                |
| --------------------- | ---- | ------- | ------------------- |
| **Smart Connections** | 连接器  | 侧边栏被动展示 | 发现笔记间隐形关联、语义搜索      |
| **Copilot**           | 助理   | 聊天面板对话  | RAG 问答、笔记检索、选中文本操作  |
| **Smart Composer**    | 编辑器  | 编辑器内联操作 | 情境感知写作、一键编辑、Diff 应用 |
| **Text Generator**    | 写作助手 | 光标处内联生成 | 续写、改写、Prompt 模板     |

**关键设计亮点：**

* **RAG（检索增强生成）**：Copilot 和 Smart Connections 都能基于整个笔记库做语义检索，回答时标注引用来源

* **`@`** **引用文件**：Smart Composer 在聊天框输入 `@<filename>` 选择特定文件作为上下文

* **`/`** **调用 Prompt 模板**：Smart Composer 支持 `/` 调出预设模板

* **一键应用 Diff**：AI 建议修改后，用户可一键 Apply 或 Reject，类似 Cursor

* **MCP 协议支持**：Smart Composer 支持连接外部 MCP 服务器

* **本地模型**：通过 Ollama 支持完全离线的本地 LLM

* **Prompt 模板变量**：支持 `{{selection}}`、`{{title}}` 等变量

### 1.3 Notion AI

Notion AI 是**深度内置**的官方功能，2026 年已发展为全面的 AI 生产力平台：

**触发方式（三种）：**

| 方式   | 触发操作         | 适用场景      |
| ---- | ------------ | --------- |
| 空格键  | 新行按 Space    | 从零起草      |
| 选中文本 | 高亮 -> Ask AI | 改善现有内容    |
| 斜杠命令 | 输入 `/AI`     | 生成摘要、提取洞察 |

**核心 AI 能力：**

| 能力                | 描述                      |
| ----------------- | ----------------------- |
| **内联写作**          | 起草、续写、改写，结果就地渲染         |
| **预设指令**          | 改进写作、总结、翻译、改变语气、加长/缩短   |
| **Diff 预览**       | 编辑建议显示删除线+新文本，可接受/放弃    |
| **@-mention 上下文** | 通过 @引用其他页面作为 AI 上下文     |
| **AI 块**          | 可配置的 AI 生成块，指定数据来源和输出格式 |
| **50 页上下文窗口**     | 跨 50 个页面的深层上下文理解        |
| **跨页面感知**         | AI 块可解析链接页面的内容          |
| **自动填充**          | 数据库属性 AI 自动填充（<3秒）      |
| **语音输入**          | macOS/Windows 语音输入提示词   |
| **自定义 Prompt 模板** | 工作区级别可重用模板              |
| **智能建议**          | 无需手动提示的主动写作建议           |
| **Workers**       | 沙盒环境执行 JS/Python 代码     |

**Notion AI 的输出交互设计：**

* **生成状态**：显示 "Reading page" / "Editing" / "Thinking" + 动态动画

* **结果操作条**：`[插入下方]` `[重试]` `[关闭]`

* **Diff 视图**：原文删除线 + 新文本高亮，`[接受]` `[放弃]`

* **聊天回复**：引用来源 + 点赞/踩 + "添加到页面" + "复制"

## 3. 方案验证结论

### 3.1 你的方案已覆盖的优势点

1. **Mermaid/LaTeX 专项指令** -- 这是 LanisMD 的差异化亮点，三家竞品都没有做到这个级别的专项支持
2. **流式 SSE + 就地渲染** -- 与行业最佳实践一致
3. **OS Keychain 存储** -- 安全性优于 Typora 插件（配置文件）和 Obsidian 插件（localStorage）
4. **Rust 后端发请求** -- Key 不进前端内存，这一点做得最好
5. **错误分类** -- 6 种错误码 + 友好文案，考虑周到
6. **取消机制** -- CancellationToken 方案成熟

### 3.2 建议补充的能力（按优先级排序）

#### P0 - 强烈建议加入（竞品普遍具备，用户期望高）

| 能力                 | 竞品参考                                              | 建议做法                                                                                    | 影响范围         |
| ------------------ | ------------------------------------------------- | --------------------------------------------------------------------------------------- | ------------ |
| **Diff 预览（接受/放弃）** | Notion（删除线+新文本）、Obsidian Smart Composer（一键 Apply） | `replace-selection` 模式不直接替换，而是先展示 Diff 视图，用户确认后再替换。用 ProseMirror Decoration 渲染删除线+高亮新文本 | ai-inline 插件 |
| **重试按钮**           | 三家都有                                              | 你已规划在 Phase 3，建议提前到 Phase 2，这是基本交互                                                      | ai-inline 插件 |
| **整篇上下文润色**        | typora-ai-edit 的"With Context"模式                  | `/polish` 除了处理选中文本外，增加一个选项将光标前后 N 段作为 context 注入 prompt，让润色更准确                          | commands.ts  |

#### P1 - 建议考虑（明显提升体验）

| 能力                             | 竞品参考                                         | 建议做法                                                | 影响范围                     |
| ------------------------------ | -------------------------------------------- | --------------------------------------------------- | ------------------------ |
| **自定义 Prompt / System Prompt** | typora-ai-edit、Obsidian 全系                   | 在设置页增加一个"自定义系统提示词"文本框，全局生效。Phase 3 再做自定义指令          | AiSettings + commands.ts |
| **模型快速切换**                     | typora-ai-edit（右键菜单）、Obsidian Copilot（对话中切换） | 在 AI 菜单或浮层中增加"当前模型"指示 + 一键切换，无需进设置页                 | slash-menu + tooltip     |
| **改变语气/风格**                    | Notion AI（预设：专业/随意/直接/友好）                    | 为 `/polish` 增加子选项：专业、简洁、友好、学术，作为 prompt 参数          | commands-ai.ts           |
| **缩短/扩展**                      | Notion AI（Make shorter/Make longer）          | 新增 `/shorter` 和 `/longer` 两个轻量指令，或作为 `/polish` 的子选项 | commands-ai.ts           |

#### P2 - 值得关注（差异化竞争力，但投入较大）

| 能力                | 竞品参考                                   | 评估                                                                |
| ----------------- | -------------------------------------- | ----------------------------------------------------------------- |
| **Ghost Text 补全** | typora-copilot                         | 体验极好但实现复杂，需要 ProseMirror 层面的幽灵文本 Decoration + 节流请求。建议作为独立 Phase 4 |
| **图表迭代修改**        | typora-ai-edit                         | 生成 Mermaid 后支持追问修改。需要对话历史管理，可考虑 Phase 3                           |
| **图像分析**          | typora-ai-edit                         | 右键图片 -> AI 描述。需要多模态模型支持（智谱 GLM-4V），技术可行但需额外模型配置                   |
| **联网搜索**          | typora-ai-edit（Tavily）、Obsidian、Notion | 需要额外的搜索 API Key，增加用户配置负担。建议 Phase 4+                              |
| **自定义指令模板**       | Obsidian（`/` + 模板变量）、Notion（工作区模板）     | Phase 3 已规划，优先级合理                                                 |

#### 不建议当前阶段考虑

| 能力           | 原因                                            |
| ------------ | --------------------------------------------- |
| RAG/笔记库检索    | LanisMD 是单文件编辑器定位，非知识库工具；方案文档 1.2 已明确"不做 RAG" |
| MCP 协议集成     | 方案文档 1.2 已明确"不做 MCP"，当前阶段过重                   |
| 本地模型（Ollama） | 增加部署复杂度，与"轻量桌面编辑器"定位冲突                        |
| Workers/代码执行 | Notion 特有的平台化能力，不适合编辑器                        |
| 语音输入         | 投入产出比低，依赖系统语音识别                               |

## 4. 交互设计对比与建议

### 4.1 触发入口对比

```
LanisMD（当前方案）:
  / -> AI 助手 -> 子菜单        （需要 2-3 步）
  选中文本 -> 浮层 [AI] -> 子菜单 （需要 2-3 步）

Typora (ai-edit):
  Cmd+E                          （1 步直达问答）
  右键 -> AI Optimize             （2 步）

Obsidian (Text Generator):
  Ctrl+J                          （1 步直达生成）

Notion AI:
  Space                           （1 步，新行时）
  选中 -> Ask AI                   （2 步）
  /AI                              （1 步）
```

**建议**：考虑为最常用的指令（如润色）增加一个全局快捷键（如 `Cmd+Shift+I`），直接触发 AI 面板，减少操作步骤。

### 4.2 输出交互对比

```
LanisMD（当前方案）:
  生成中: 占位块 + 流式追加 + [取消]
  完成后: [接受] [重试] [丢弃]    -- Phase 3 才有重试

竞品最佳实践 (Notion):
  生成中: "Thinking..." + 流式追加 + [Stop]
  完成后: Diff 视图（删除线+新文本）+ [Accept] [Retry] [Discard]
  追加:   输出下方显示后续输入框，支持"让它更短""换个语气"

竞品最佳实践 (typora-ai-edit):
  完成后: 直接替换 + 支持 Undo
```

**建议**：

1. Phase 1 的 `[接受] [重试] [丢弃]` 操作条应该同时上线（重试不应延后到 Phase 3）
2. 对于 `replace-selection` 模式，先展示 Diff 再替换，比直接替换安全得多
3. 完成后的操作条下方可增加一个小输入框："继续调整..."，支持追问式迭代

## 5. 总结：你的方案评分

| 评估维度 | 评分    | 说明                                                   |
| ---- | ----- | ---------------------------------------------------- |
| 功能覆盖 | 8/10  | 7 个指令覆盖主要场景，Mermaid/LaTeX 是亮点。缺 Diff 预览、语气调整、上下文润色   |
| 交互设计 | 7/10  | 斜杠+浮层两个入口合理，但缺快捷键直达和 Diff 预览                         |
| 技术架构 | 9/10  | Rust SSE + Keychain + CancellationToken，安全性和性能都是最佳实践 |
| 安全隐私 | 10/10 | Key 不进前端、不记录内容日志、Keychain 存储，三家中最优                   |
| 阶段规划 | 8/10  | 三阶段合理，建议重试和 Diff 提前到 Phase 2                         |
| 差异化  | 8/10  | Mermaid/LaTeX 专项 + 三家国内服务商预设，定位清晰                    |

### 最终建议的方案调整

1. **Phase 1 调整**：把"重试按钮"从 Phase 3 提到 Phase 1（只需保留上次 prompt 即可，成本极低）
2. **Phase 2 新增**：Diff 预览模式（`replace-selection` 时先展示对比再确认）
3. **Phase 2 新增**：为 `/polish` 增加语气子选项（专业/简洁/友好/学术）
4. **Phase 2 新增**：增加一个 AI 全局快捷键（如 `Cmd+Shift+I`）
5. **Phase 2 新增**：模型快速切换（菜单内切换，不用进设置页）
6. **Phase 3 新增**：自定义 System Prompt（设置页文本框）
7. **Phase 3 新增**：输出后"继续调整"输入框（支持追问式迭代）
